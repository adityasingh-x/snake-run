A proposed implementation plan for the next milestone is available in:

- plans/ACTIVE.md

Perform a Staff Engineer level review of the plan.

Follow AGENTS.md and read any additional project documentation required by AGENTS.md and the task.

Requirements:

- Do not modify plans/ACTIVE.md.
- Do not create a replacement plan.
- Create a review document only.
- Save the review as plans/PLAN_REVIEW.md.

Review Objectives:

1. Verify alignment with ROADMAP.md.
2. Verify alignment with AGENTS.md.
3. Verify alignment with ARCHITECTURE.md.
4. Identify missing implementation details that would block another AI agent.
5. Identify scope creep or future-milestone leakage.
6. Identify unnecessary complexity.
7. Identify missing risks, assumptions, or verification steps.
8. Identify documentation updates that may have been overlooked.
9. Evaluate whether the plan can be handed off to another AI agent with minimal ambiguity.

The review should contain:

# Overall Assessment

- Strengths
- Weaknesses
- Major Risks
- Recommended Changes

# Detailed Findings

For each finding:

- Severity (Critical / High / Medium / Low)
- Description
- Recommendation

# Handoff Assessment

Evaluate:

- Phase structure
- Task decomposition
- Verification strategy
- Definition of Done
- AI-agent execution readiness

# Final Recommendation

Choose one:

- Approve
- Approve with Minor Changes
- Approve with Major Changes
- Reject

Favor:

- simplicity
- maintainability
- repository alignment
- executable plans

Do not recommend complexity unless it is clearly justified by the milestone goals.
